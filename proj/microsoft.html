UX Design Internship

Role

UX Design Intern

Duration

Dec 2020 - Jun 2021

Team

Azure Apps and Infrastructure Team, Michael Shao (Manager)

Challenge

Creating a more intuitive no-code chatbot builder for business administrators running Microsoft Azure and Cognitive Services.

Outcome

A redesigned, drag-and-drop builder with a new visual identity and user experience. Jump to the final designs here!

Impact

Overview

I have permission from Microsoft to showcase this project. A free demo can be requested when you contact our customer engineers at cloudcrew@microsoft.com.

Operational communication is time-consuming, and a FAQ page is often not sufficient. Therefore, we designed Bot Portal, a streamlined chatbot builder that is more compact than Power Virtual Agents, a web app that builds complex conversation flows. The team has already built a beta version of Bot Portal, and I was responsible for creating the next iteration.

As the only designer on the team, I led the web app's end-to-end UI/UX design. I also redesigned the messaging and security controls, but this case study focuses on building a conversation flow.

Problem

How do we create an intuitive no-code chatbot builder for operational business administrators with no knowledge of programming?

Through user interviews, I discovered some difficulties of an operational manager trying to input information into the current system.

Pain Points

  1. Choice Overload: There are 14 response nodes for users to choose from, easily overwhelming users building simple flows with few responses.
  2. Unintuitive Interaction: The user must build flows by creating new if/or/and statements, then using indices to represent multiple choices. Tldr. It's too hard to use.
  3. Interrupted flows: Text and file editors all exist as modals currently, which often interrupts the user's actions and disables the main content.
  4. Repetitive controls: Same response nodes have to be recreated each time, which is time-consuming.
  5. Lack of brand identity: The look doesn't feel "Microsoft."

Project Goals

So, how do we streamline the chatbot-building experience?

  1. Create non-programmer friendly flow-building interactions
  2. Prioritize frequent actions and restructure information architecture
  3. Enable users to save common nodes
  4. Underscore Microsoft's brand identity

Process

Information Architecture

Through card sorting, I restructured the web app's main navigation to prioritize the chatbot builder and decrease security controls' visual importance. Within the chatbot builder, I conducted card sorting again and categorized actions under user motivations, which helped me reduce the action nodes from 14 to 7 and resolve choice overload.

Competitors' UX Patterns

By performing competitive analysis, I decided on a drag-and-drop interaction with a tree structure display.Users without a programming background can build a conversation quickly and understand the flow clearly from the tree structure.

Code-like structure

Adding onto branches

Our choice: Drag and drop!

Product Requirement Document (PRD)

Since there were no PMs assigned to this project, both the developer and I stepped into the roles of a PM. I proactively wrote a PRD to reduce the chance of miscommunication and record our thoughts on each iteration.

Exploring and Testing

I created multiple interactive prototypes to test edge cases and compound use cases. This guided our critiques and evaluation process, which also prompted new features to be included in the final version.

New Feature 1: Adding customized intents

New Feature 2: Storing multiple languages in each node, so that the Chatbot responses in the user question's language

Visual Design

Since we decided to redesign the entire visual identity, I also provided a UI Style Guide to lay out all the components and their states, guiding front-end development in Angular.

Final Design

Check out Bot Portal's major redesigns!

Decision 1

Build flows quickly

Users can simply drag and drop nodes from the toolbox to build the conversation flow.

Decision 2

Test conversations easily

Pressing the "Test" button allows the user to efficiently test out the flow before syncing it to the real bot.

Decision 3

Save common nodes

The user can save common nodes as "global nodes." When they edit the main instance in the Global Node Editor, instances of this node in all conversation flows are updated.

Decision 4

Edit details in the sidebar

To reduce modals on all action menus that interrupt the user's flow, I introduced the sidebar. It effectively hides complicated actions and creates a consistent place to edit global nodes.

Impact

Despite facing many technical and communication challenges, I was super proud of leading the design of this new iteration, prototyping almost 200 screens!

Takeaways

I grew tremendously as a designer, communicator, and team player during this internship. Some key lessons include:

1. Presentation and documentation

While I was only collaborating with one other engineer, it was crucial for me to ask questions concisely to make sure my design choices were clear because we mostly worked remotely. Since this was a major redesign, I saw the necessity to write a PRD so a new designer can easily pick up where I finished.

2. Catching edge cases

I approached problems with a user-centric focus while the engineers came from an implementation perspective, which was intriguing because we caught different edge cases due to our differing viewpoints.

3. Getting feedback early

Demonstrating the functionality through prototypes better represented the user flows I was envisioning, which made the critique process easier. I also seeked out designers on other teams to analyze the new user's experience to create informed and guided interactions.
